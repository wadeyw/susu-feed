### 1) Prompt injection is not a bug — it's a fundamental model architecture problem
- Track: better_decisions
- Why it matters to me: This reframes everything I thought I knew about AI security. The research by Ye, Cui, and Hadfield-Menell shows that models don't actually distinguish between privileged system text and untrusted user input by content — they distinguish by *style*. "Destyling" text (rewriting it in a slightly different format so it looks less like the expected role-tag format) plunges attack success from 61% to 10%. This means prompt injection isn't a solvable engineering problem; it's a consequence of how LLMs process text as stylistic patterns rather than semantic containers.
- Tradeoff / tension: If defense requires genuine role perception that models don't have, then every production system using LLMs with user-supplied text carries a persistent injection surface. The alternative — sandboxing, strict input validation, model isolation — adds latency, cost, and complexity. There is no clean solution.
- Implication: Any system I build that takes user input and passes it to an LLM (even with system prompts) should assume injection is possible. Architecture choices like output validation, separate model instances for trusted vs untrusted roles, and explicit delimiting become non-negotiable, not optional.
- Action: Watch. When designing any AI feature that processes user content, build injection resistance into the architecture, not the prompt.
- Confidence: high
- Source: Simon Willison — https://simonwillison.net/2026/Jun/22/prompt-injection-as-role-confusion/

### 4) California AB 2047: regulating 3D printers before the detection technology even exists
- Track: daughters_future
- Why it matters to me: California is advancing legislation that would ban students, educators, and businesses from using 3D printers unless manufacturers install "firearm detection" technology — a technology that the state's own DOJ is still studying and hasn't certified. The bill passed the Assembly and heads to Senate Judiciary. 29 organizations oppose it, including Prusa Research, Maker Faire, and the RepRap founder. This is a template for how regulation can restrict access to general-purpose tools based on fear of misuse, using technology that doesn't exist yet, with penalties of $25,000 per violation.
- Tradeoff / tension: Public safety concerns about 3D-printed firearms are real, but the proposed solution targets legitimate users (educators, makers, small businesses) while bad actors will easily bypass any detection system. The bill also raises First Amendment issues since CAD files are protected speech. The contradiction: the manufacturer mandate references certification rules that were deleted after 33 amendments.
- Implication: This pattern — regulate access to foundational tools before the technical basis for regulation exists — will repeat. If daughters grow up in an environment where general-purpose fabrication tools are restricted for students, it shapes what they can learn to build. The broader signal: regulatory risk for maker education, hardware entrepreneurship, and any domain where tools can be dual-use.
- Action: Watch. Track whether this bill advances in the Senate committees (Judiciary and Public Safety). If it passes, it will likely face legal challenges and could become a model for other states. For family planning, consider what hands-on fabrication and maker education access looks like in different jurisdictions.
