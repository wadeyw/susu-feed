### 3) Better Models: Worse Tools — SOTA models regress on non-standard tool schemas

- Track: better_decisions
- Why it matters to me: Armin Ronacher's report (via Simon Willison) shows that newer, more capable Claude models (Opus 4.8) are actually making *more* tool-use errors on non-standard schemas than smaller models. This contradicts the linear assumption that "bigger model = better at everything." It's a concrete failure mode for anyone building agent systems. Connects to AQ4 (failure modes and decision hygiene).
- Tradeoff / tension: As models get stronger on standard benchmarks, their confidence in non-standard or poorly documented interfaces increases, leading them to invent fields and arguments that don't exist. The tradeoff is between using the strongest available model (which may hallucinate tool schemas) versus a slightly weaker model that follows conventions more faithfully. This is a reliability-versus-capacity tradeoff that doesn't show up in benchmark charts.
- Implication: If you're building agent systems, the model strength you pick directly affects your schema design burden. Stronger models require stricter schema enforcement, not looser. This suggests that for production agent systems, you may want to decouple reasoning strength from tool-use reliability — perhaps using a smaller model for tool invocation and reserving the big model for pure reasoning tasks.
- Action: Act — if you have any agent systems in flight, audit tool-call schemas against what your current model actually produces, not what the docs say it should produce. Test with non-standard schemas specifically.
- Confidence: high
- Source: Simon Willison (quoting Armin Ronacher) — https://simonwillison.net/2026/Jul/4/better-models-worse-tools/
